Architecture
============

Cette section documente les **décisions d'architecture** structurantes de
``baobab-auth-core`` sous forme d'*Architecture Decision Records* (ADR).

Chaque ADR capture une décision, son contexte, ses conséquences et les
alternatives écartées. Les ADR sont **immuables** : une décision révisée fait
l'objet d'un nouvel ADR qui supersède le précédent (jamais d'édition rétroactive
du fond).
