# US-050 — Stabiliser les contrats publics et préparer l'intégration

| Champ | Valeur |
|-------|--------|
| Statut | Implémentée (v0.5.0) |
| Version cible | v0.5.0 |
| Origine | Cahier des charges `baobab-auth-core` v0.5.0 |

## Récit

En tant qu'équipe préparant l'intégration des briques `database`, `security`,
`api`, `client` et `admin`, je veux des contrats publics stables (exports,
DTO, codes d'erreur, ports, cas d'usage de lecture/admin) et des tests
contractuels, afin d'intégrer sans réécrire le core ni y ajouter d'infrastructure.

## Definition of Done (US)

API publique documentée, DTO sans secret, exceptions avec `error_code`/`safe_message`,
contrats documentés, cas d'usage de lecture et admin disponibles, tests contractuels
présents, aucune dépendance interdite, `ruff`/`mypy`/`pytest` OK, couverture ≥ 95 %.
**Atteinte** au tag `v0.5.0`.
